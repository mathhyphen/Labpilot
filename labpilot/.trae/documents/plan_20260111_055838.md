## 将企业微信通知替换为钉钉群聊机器人通知

### 1. 问题分析
当前项目使用企业微信应用和机器人进行通知，但用户希望改用钉钉群聊机器人。需要修改所有与企业微信相关的代码，替换为钉钉机器人的实现。

### 2. 主要修改内容

#### 2.1 修改 `labpilot/notify.py` 文件
- 将 `WecomNotifier` 类替换为 `DingTalkNotifier` 类
- 实现钉钉机器人的通知发送逻辑
- 支持钉钉群聊机器人的 Markdown 消息格式
- 优化错误处理和配置验证

#### 2.2 修改 `labpilot/cli.py` 文件
- 删除企业微信相关的函数（`_get_wecom_access_token`）
- 修改 `send_wecom_notification` 函数为 `send_dingtalk_notification`
- 更新所有通知调用为使用钉钉通知
- 修改默认配置为钉钉机器人配置

#### 2.3 修改 `config.yaml` 文件
- 将企业微信配置替换为钉钉机器人配置
- 添加钉钉机器人的配置说明和示例

#### 2.4 修改 `USAGE.md` 文件
- 将企业微信相关的文档替换为钉钉机器人的配置说明
- 添加钉钉机器人的创建和配置步骤
- 更新常见问题和最佳实践

### 3. 实现步骤

1. **修改 `labpilot/notify.py` 文件**：
   - 创建 `DingTalkNotifier` 类，实现钉钉机器人通知功能
   - 优化错误处理和配置验证
   - 支持钉钉机器人的 Markdown 消息格式

2. **修改 `labpilot/cli.py` 文件**：
   - 删除企业微信相关函数
   - 更新通知发送函数
   - 修改默认配置
   - 更新所有通知调用

3. **修改 `config.yaml` 文件**：
   - 更新配置结构为钉钉机器人配置
   - 添加详细的注释和示例

4. **修改 `USAGE.md` 文件**：
   - 更新通知配置文档
   - 添加钉钉机器人创建和配置步骤
   - 更新常见问题和最佳实践

5. **测试修改**：
   - 运行代码检查，确保没有语法错误
   - 验证配置结构的正确性

### 4. 预期效果
- 项目不再依赖企业微信，改为使用钉钉群聊机器人
- 用户可以通过简单的配置，将实验通知发送到钉钉群聊
- 保持与原企业微信通知相同的功能和用户体验
- 提供详细的文档和示例，便于用户配置和使用
